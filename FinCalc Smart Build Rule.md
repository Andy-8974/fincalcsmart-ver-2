FinCalc Smart Build Rule

We are building FinCalc Smart as a financial tools platform, not just a calculator website.

Workflow:
1. Lock the foundation first:
   - Navigation
   - Country-aware routing
   - Search behavior
   - Calculator metadata
   - Footer/menu consistency
   - V2 calculator template rules

2. Build Phase 1 calculators one by one using the locked V2 template.

3. Treat AI Analysis / AI Insights as the product differentiator.

4. Do not redesign the foundation during calculator builds.

5. Delay larger expansion features until the core platform is solid:
   - AI Insights page
   - Financial Guides hub
   - Save/share
   - Helpfulness signals
   - Recently used calculators

Main risk:
Scope creep, not coding.

Decision rule:
Before every change, ask:
“Does this make the whole FinCalc Smart product stronger, or only fix one small screen?”
